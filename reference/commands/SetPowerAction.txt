Description
    Sets power action of managed system.
Required Arguments
    --action    <action>
          Sets power action with:
          0 = up
          1 = down
          2 = cycle
          3 = reset
          4 = softshutdown
          5 = reboot
          6 = accycle
          7 = nmi
          8 = on
Optional Arguments
    --interval    <time interval>
          Set power cycle interval in seconds.
    --post_complete
          Waits for the managed system POST complete after rebooting.
    --power_on
          Power on the system after AC cycle.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    No
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c SetPowerAction --action up
  In-Band
    # ./saa -c SetPowerAction --action down
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c SetPowerAction --action reset
  Remote In-Band
    # ./saa -I Remote_INB --oi 192.168.34.57 --ou root --op 111111 -c
        SetPowerAction --action 2
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c SetPowerAction --action 3
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c SetPowerAction --action up
  Remote In-Band
    # ./saa -I Remote_INB -l IP_ADDR_RANGE.txt -c SetPowerAction --action 2
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c SetPowerAction --action 3
Notice
    1. Note: 'up' maps to the Redfish reset action 'ForceOn', and 'on' maps to
    Redfish reset action 'On'. Please refer to the SAA user guide for more
    detail.
    2. Note: The --power_on option is only supported by the AcCycle action, is
    not supported for in-band usage, and is required when using --post_complete.
