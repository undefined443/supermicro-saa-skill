Description
    Updates Midplane SBB CPLD.
Required Arguments
    --file    <file name>
          Updates the Midplane SBB CPLD with the given CPLD image file.
    --reboot
          Forces the managed system to reboot or power up after operation.
    --index    <index>
          Sets CPLD index.
Optional Arguments
    --individually
          Updates each Midplane SBB CPLD with its corresponding image file
          individually.
    --post_complete
          Waits for the managed system POST complete after rebooting.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    No
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c UpdateMidplaneSbbCpld --file
        CPLD.jed --index 1 --reboot --post_complete
  In-Band
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c UpdateMidplaneSbbCpld --file
        CPLD.jed --index 1 --reboot
  Remote In-Band
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c UpdateMidplaneSbbCpld --file CPLD.jed --index 1 --reboot
        --remote_saa /root/saa
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c UpdateMidplaneSbbCpld
        --file CPLD.jed --index 1 --reboot
  Remote In-Band
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c UpdateMidplaneSbbCpld --file
        CPLD.jed --index 1 --reboot
Notice
    1. Support only on Midplane board type system.
    2. System needs to be powered off while updating Midplane SBB CPLD FW.
