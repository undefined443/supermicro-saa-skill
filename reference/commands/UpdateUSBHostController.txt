Description
    Updates USB Host Controller firmware.
Required Arguments
    --file    <file name>
          USB Host Controller firmware file
    --dev_id    <Device ID>
          USB Host Controller device ID. Can be retrieved with
          GetUSBHostControllerInfo command.
Optional Arguments
    --reboot
          Forces the managed system to power cycle after operation.
    --cfg_file    <SDSi-agent config file>
          USB Host Controller config file
Usage Modes
    [In-Band] [Remote In-Band] [Multiple Systems Remote In-Band] 
Node Product Key Required
Examples
Single System
  In-Band
    # ./saa -c UpdateUSBHostController --file FirmwareFilePath --cfg_file
        ConfigFilePath --dev_id 0
    # ./saa -c UpdateUSBHostController --file FirmwareFilePath --cfg_file
        ConfigFilePath --dev_id 0 --reboot
  Remote In-Band
    # ./saa -I Remote_INB --oi 192.168.34.57 --ou root --op 111111 -c
        UpdateUSBHostController --file FirmwareFilePath --cfg_file
        ConfigFilePath --dev_id 0 --remote_saa /root/saa
    # ./saa -I Remote_INB --oi 192.168.34.57 --ou root --op 111111 -c
        UpdateUSBHostController --file FirmwareFilePath --cfg_file
        ConfigFilePath --dev_id 0 --reboot --remote_saa /root/saa
Multiple Systems
  Remote In-Band
    # ./saa -I Remote_INB -l IP_ADDR_RANGE.txt -c UpdateUSBHostController --file
        FirmwareFilePath --cfg_file ConfigFilePath --dev_id 0
    # ./saa -I Remote_INB -l IP_ADDR_RANGE.txt -c UpdateUSBHostController --file
        FirmwareFilePath --cfg_file ConfigFilePath --dev_id 0 --reboot
Notice
    1. If the platform includes Renesas USB Host Controller device, please
    execute "sudo make -f Makefile.drv" in "driver/renesas/USB" folder to create
    device driver, and SAA will automatically mount the device driver.
