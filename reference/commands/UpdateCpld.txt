Description
    Updates CPLD.
Required Arguments
    --file    <file name>
          Updates the CPLD with the given CPLD image file.
    --reboot
          Forces the managed system to reboot or power up after operation.
Optional Arguments
    --individually
          Updates each CPLD with its corresponding configuration file
          individually.
    --post_complete
          Waits for the managed system's POST to complete after rebooting.
    --index    <index>
          Updates the CPLD with the given index. The default value is empty, and
          the first CPLD will be updated.
    --poweroff
          Forces the managed system to power off during the update operation.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    No
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c UpdateCpld --file CPLD.bin
        --reboot
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c UpdateCpld --file CPLD.bin
        --index 2 --reboot
  In-Band
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c UpdateCpld --file CPLD.bin
        --reboot
  Remote In-Band
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c UpdateCpld --file CPLD.bin --reboot --remote_saa /root/saa
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c UpdateCpld --file CPLD.bin
        --reboot
  Remote In-Band
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c UpdateCpld --file CPLD.bin
        --reboot
Notice
    1. Support on RoT system of X12/H12 and later platforms.Support on non-RoT
    system of X13/H13 and later platforms.
    2. System needs to be powered off while updating CPLD FW.
