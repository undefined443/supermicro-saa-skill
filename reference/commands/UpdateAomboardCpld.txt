Description
    Updates AOM board CPLD.
Required Arguments
    --file    <file name>
          Updates the AOM board with the given CPLD image file.
Optional Arguments
    --reboot
          Forces the managed system to reboot or power up after operation.
    --individually
          Updates each AOM board CPLD with its corresponding image file
          individually.
    --dev_id    <Device ID>
          Updates the AOM board CPLD with the given AOM device ID.
    --post_complete
          Waits for the managed system's POST to complete after reboot.
    --index    <index>
          Updates the CPLD with the given index. The default value is empty, and
          the first CPLD on the AOM board will be updated.
    --aom_type    <aom type>
          Sets AOM type to update, the default value is "AOMboard":
          - AOMboard
          - AOM_SCM
          - AOM_PDB
          - AOM_FAN
          - AOM_M2
          - AOM_RSC
          - or other supported AOM type string.
    --poweroff
          Forces the managed system to power off during the update operation.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    SFT-DCMS-SINGLE
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c UpdateAomboardCpld --file
        CPLD.jed
  In-Band
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c UpdateAomboardCpld --file
        CPLD.jed
  Remote In-Band
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c UpdateAomboardCpld --file CPLD.jed
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c UpdateAomboardCpld --file
        CPLD.jed
  Remote In-Band
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c UpdateAomboardCpld --file
        CPLD.jed
