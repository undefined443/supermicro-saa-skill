Description
    Executes RoT-related actions.
Required Arguments
    --action    <action>
          Sets action to:
          1 = GetInfo
          2 = UpdateGolden
          3 = Recover
          4 = DownloadEvidence
Optional Arguments
    --file    <file name>
          Works with --action DownloadEvidence. Saves the BIOS evidence to a
          file.
    --overwrite
          Works with --action DownloadEvidence. Overwrites the output file.
    --reboot
          Works with --action UpdateGolden and Recover. Forces the managed
          system to reboot or power up after operation.
    --redfish
          Enables support for pure Redfish.
    --post_complete
          Waits for the managed system's POST to complete after reboot.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    No
    SFT-DCMS-SINGLE for action Recover and DownloadEvidence
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c BiosRotManage --action 3
        --reboot
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c BiosRotManage --action 4
        --file evidence.bin.gz
  In-Band
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c BiosRotManage --action
        UpdateGolden --reboot
  Remote In-Band
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c BiosRotManage --action UpdateGolden --reboot --remote_saa
        /root/saa
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c BiosRotManage --action
        GetInfo
  Remote In-Band
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c BiosRotManage --action GetInfo
Notice
    1. Support only on RoT system of X12 and later platforms.
    2. UpdateGolden action will overwrite golden image with current BIOS
    firmware.
    3. DownloadEvidence action is only available after automatic or manual BIOS
    recovery.
    4. The BIOS evidence is a compressed gzip file.
