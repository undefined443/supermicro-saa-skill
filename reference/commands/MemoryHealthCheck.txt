Description
    Sets the memory health checking function of the managed system.
Required Arguments
    --action    <action>
          Sets action to:
          1 = GetCurrentStatus
          2 = Persistent
          3 = Enable
          4 = Disable
Optional Arguments
    --reboot
          Forces the managed system to reboot or power up after operation.
    --json_view
          Sets the output format to JSON.
    --post_complete
          Waits for the managed system POST complete after rebooting.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    SFT-DCMS-SINGLE
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c MemoryHealthCheck --action
        GetCurrentStatus
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c MemoryHealthCheck --action
        Persistent --reboot
  In-Band
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c MemoryHealthCheck --action Enable
        --reboot
  Remote In-Band
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c MemoryHealthCheck --action Enable --reboot --remote_saa
        /root/saa
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c MemoryHealthCheck --action
        Disable --reboot --post_complete
  Remote In-Band
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c MemoryHealthCheck --action
        Enable --reboot --remote_saa /root/saa
Notice
    1. If the action is set to "Enable", BIOS will perform memory health check
    at the next power on, launch self-healing mechanism if memory errors occur,
    and will set the action value back to "Disable". In contrast, if the action
    is set to "Persistent", BIOS will execute the memory health check flow in
    every power on until the action is changed.
    2. As BIOS executes the memory health check flow, there will be related
    system event logs shown on the console output.
