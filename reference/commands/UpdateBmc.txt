Description
    Updates BMC.
Required Arguments
    --file    <file name>
          Updates the BMC with the given BMC file.
Optional Arguments
    --individually
          Updates each BMC with corresponding firmware file individually.
    --overwrite_cfg
          Overwrites the current BMC configuration using the factory default
          values in the given BMC image file.
    --overwrite_sdr
          Overwrites current BMC SDR data.
    --overwrite_ssl
          Overwrites current BMC SSL configuration. (Only supported by the
          X12/H12 and later platforms except the H12 non-RoT systems.)
    --redfish
          Enables support for pure Redfish.
    --backup
          Backs up the current BMC image. (Only supported by the RoT systems.)
    --forward
          Confirms the Rollback ID and upgrades to the next revision.(Only
          supported by the X12/H12 and later platforms except the H12 non-RoT
          systems.)
    --bmc_boot_check
          Checks if BMC boots up within 16 minutes after update. (Only supported
          by the X12/H12 and later platforms except the H12 non-RoT systems.)
    --upgrade_only
          Firmware updates are only performed when the version is newer.
    --check_reboot_required
          Displays a warning message when a system reboot or power cycle is
          required.
    --no_waiting
          Bypasses the standard waiting period after BMC firmware updates.
          Normally the tool waits for the BMC to complete its cold reset cycle
          before proceeding. This option skips the wait time but may cause
          connection failures if the BMC is still rebooting.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    No
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c UpdateBmc --file BMC.bin
  In-Band
    # ./saa -c UpdateBmc --file BMC.bin
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c UpdateBmc --file BMC.bin
  Remote In-Band
    # ./saa -I Remote_INB --oi 192.168.34.57 --ou root --op 111111 -c UpdateBmc
        --file BMC.bin
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c UpdateBmc --file BMC.bin
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c UpdateBmc --file BMC.bin
  Remote In-Band
    # ./saa -I Remote_INB -l IP_ADDR_RANGE.txt -c UpdateBmc --file BMC.bin
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c UpdateBmc --file BMC.bin
Notice
    1. -I Redfish_HI is required for in-band usage on H12/X12 and later
    platforms except H12 non-RoT systems.
    2. For Rollback ID increases case, option --bmc_boot_check will check BMC is
    booted up in 24 minutes after update.
