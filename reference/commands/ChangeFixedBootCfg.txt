Description
    Updates the BIOS fixed boot order.
Required Arguments
    --file    <file name>
          Updates the BIOS fixed boot order with the given configuration file.
Optional Arguments
    --reboot
          Forces the managed system to reboot or power up after operation.
    --individually
          Updates each BIOS fixed boot configuration with corresponding
          configuration file individually.
    --post_complete
          Waits for the managed systemâ€™s POST to complete after reboot.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    SFT-DCMS-SINGLE
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c ChangeFixedBootCfg --file
        FixedBootCfg.xml
  In-Band
    # ./saa -I Redfish_HI -u ADMIN -p ADMIN -c ChangeFixedBootCfg --file
        FixedBootCfg.xml
  Remote In-Band
    # ./saa -I Remote_RHI --oi 192.168.34.57 --ou root --op 111111 -u ADMIN -p
        ADMIN -c ChangeFixedBootCfg --file FixedBootCfg.xml
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c ChangeFixedBootCfg --file
        FixedBootCfg.xml
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c ChangeFixedBootCfg --file
        FixedBootCfg.xml --individually
  Remote In-Band
    # ./saa -I Remote_RHI -l IP_ADDR_RANGE.txt -c ChangeFixedBootCfg --file
        FixedBootCfg.xml
Notice
    1. --individually option should be used with -l option for concurrent OOB
    usage.
