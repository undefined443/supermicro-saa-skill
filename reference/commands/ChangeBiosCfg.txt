Description
    Updates the BIOS configuration.
Required Arguments
    --file    <file name>
          Updates the BIOS with the given configuration file.
Optional Arguments
    --current_password    <current password>
          Checks the current BIOS Administrator password.
    --reboot
          Forces the managed system to reboot or power up after operation.
    --individually
          Updates each BIOS with corresponding configuration file individually.
    --skip_unknown
          Skips the unknown settings or menus in the BIOS configuration file.
    --skip_bbs
          Skips the BBS-related menus in the BIOS configuration file.
    --post_complete
          Waits for the managed system POST complete after rebooting.
    --cur_pw_file    <Current password file>
          The specified file path to read current password.
Usage Modes
    [OOB] [In-Band] [Multiple Systems OOB] [Remote In-Band] [Multiple Systems
    Remote In-Band] 
Node Product Key Required
    Yes
    SFT-DCMS-SINGLE for some configuration items
Examples
Single System
  OOB
    # ./saa -i 192.168.34.56 -u ADMIN -p ADMIN -c ChangeBiosCfg --file
        current.file --reboot
  In-Band
    # ./saa -c ChangeBiosCfg --file current.file --reboot
  Remote In-Band
    # ./saa -I Remote_INB --oi 192.168.34.57 --ou root --op 111111 -c
        ChangeBiosCfg --file current.file --reboot --remote_saa /root/saa
Multiple Systems
  OOB
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c ChangeBiosCfg --reboot
    # ./saa -l IP_ADDR_RANGE.txt -u ADMIN -p ADMIN -c ChangeBiosCfg
        --individually --reboot
  Remote In-Band
    # ./saa -I Remote_INB -l IP_ADDR_RANGE.txt -c ChangeBiosCfg --individually
        --reboot
Notice
    1. The --skip_unknown and --skip_bbs options are only supported on Intel(R)
    Xeon(R) Scalable Processors with Intel(R) C620 Series Chipsets and later
    platforms.
    2. Some configuration tags are "<LicenseRequirement>" in BIOS configuration
    XML file and require SFT-DCMS-SINGLE node product key.
    3. --individually option should be used with -l option for multiple OOB
    usage.
